# sketchbook

This repository is a software sketchbook: a single place to preserve small
frontend, backend, database, algorithm, Java, C, and distributed systems
experiments while gradually turning useful pieces into teaching material.

It is intentionally a notebook, not a polished product. Originals belong in
`archive/originals/` as historical artifacts. Cleaned versions, lessons, demos,
and exercises can be built from them over time.

## How To Read This Repo

1. Start with a chapter in `frontend/` or `backend/`.
2. Read the chapter README to see the concept progression.
3. Inspect matching project metadata in `data/projects.json`.
4. Compare the archived original with the cleaned target folder.
5. Turn useful parts into a lesson, demo, exercise, or reference example.

## Main Areas

- `frontend/` keeps cleaned teaching spaces for HTML/CSS, Bootstrap,
  JavaScript, React, and UI patterns.
- `backend/` keeps cleaned teaching spaces for Java, C, databases, algorithms,
  distributed systems, and API patterns.
- `lessons/` contains concept-first teaching notes.
- `demos/` contains runnable examples used to explain a concept.
- `exercises/` contains student practice tasks.
- `archive/originals/` preserves old projects before cleanup.
- `data/` contains project, chapter, and tag metadata used by `index.html`.
- `docs/` explains the philosophy, teaching model, import map, and maintenance
  rules.

Open `index.html` through a local web server to browse the project metadata.
