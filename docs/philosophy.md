# Philosophy

This repository is a software sketchbook.

It collects small frontend, backend, database, algorithm, and systems
experiments. The goal is to preserve experiments while gradually turning useful
pieces into lessons, demos, exercises, and standalone projects.

The sketchbook is intentionally rough at first. It values preservation, clarity,
and progressive learning over polish.

Original projects are kept as historical artifacts. Cleaned versions can be
built gradually from them.

## What Belongs Here

- Small experiments that show one or two concepts clearly.
- Old student or practice projects that can become teaching examples.
- Lesson notes that explain concepts before showing code.
- Demos that make behavior visible.
- Exercises that ask students to modify or extend existing code.

## What Does Not Belong Here

- Secrets, credentials, cookies, browser profiles, or private keys.
- Polished product code mixed into experiments without clear labels.
- Rewrites that erase the original before a clean version exists.
