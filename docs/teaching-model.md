# Teaching Model

The sketchbook is organized concept-first. Projects are raw teaching seeds, not
final curriculum by default.

The basic flow is:

```text
Concept -> Small explanation -> Demo -> Code reading -> Modification exercise -> Optional extension
```

## Progressive Learning Paths

Each chapter should introduce concepts in small steps, then reinforce them with
projects and exercises. A teacher should be able to say: start here, understand
this concept, inspect this demo, then modify this exercise.

## Sketches As Teaching Seeds

Old projects are useful because they show real attempts, rough edges, repeated
mistakes, and practical constraints. They should be preserved first, then
cleaned into focused versions only when there is a teaching reason.

## Demos And Exercises

Demos make an idea visible. Exercises turn the idea into practice. A polished
demo should usually have a short lesson page, and every exercise should state
the concepts students need before starting.

## Examples

HTML & CSS:

```text
tags -> layout -> cards -> navigation -> small page
```

JavaScript:

```text
variables -> DOM selection -> events -> state -> small interaction
```

Java:

```text
classes -> objects -> collections -> files -> small application
```

Databases:

```text
tables -> keys -> relationships -> queries -> small interface
```

Algorithms:

```text
arrays -> searching -> sorting -> graphs -> route finding
```

Distributed Systems:

```text
processes -> communication -> synchronization -> concurrency -> small lab
```
